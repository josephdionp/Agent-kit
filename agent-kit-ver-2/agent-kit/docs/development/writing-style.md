# Writing style

Use simple, efficient English.

Rules:

- Keep wording short and clear.
- Prefer direct sentences.
- Remove repeated meaning.
- Rephrase long text without changing meaning.
- Use simple terms before technical terms.
- Keep structure easy to scan.
- Do not add filler.
- Do not make documents longer than needed.
- Keep the original meaning when reducing text.

For code and command references:

- Exact, copy-pasteable commands. No prose describing what a command does
  when the command itself makes it clear.
- Paths in backticks. File-line refs as `file.ts:42`.
