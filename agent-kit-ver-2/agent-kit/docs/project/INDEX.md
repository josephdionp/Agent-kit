# Project

Product goals, technology choices, and cross-cutting decisions. Stable —
edited only on a ratified decision. On any conflict, the decision register
wins.

- [`overview.md`](overview.md) — product name, purpose, users, non-goals
- [`stack.md`](stack.md) — language, runtime, frameworks, tools
- [`architecture.md`](architecture.md) — system design, boundaries, layers
- [`decisions/`](decisions/) — architecture decision records (ADRs)

If any of these grows past ~500 lines, split into a folder of numbered
chapters with the parent `.md` as a router. See
`docs/development/documentation-rules.md`.
